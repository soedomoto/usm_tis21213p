Apa itu Cookies?

Cookies adalah beberapa text sederhana yang dikirimkan oleh website yang dikunjungi
Cookies dapat membantu mengingat beberapa informasi dari website yang dikunjungi
Cookies mempermudah akses ke website ketika berpindah halaman, atau dikunjungi ulang

Sifat Cookies

Cookies tidak boleh melebihi 4KB text dalam form key=value data
Cookies disimpan oleh browser
Cookies bersifat spesifik pada domain yang mengirimkannya
Artinya kita hanya dapat membaca, mengubah, menghapus cookies pada domain yang mengirimkannya.
Cookies bersifat spesifik pada browser yang menyimpannya
Artinya kita tidak dapat membaca, mengubah, menghapus cookies pada browser yang berbeda.

Men-set cookies

Cookies dapat di-set dari server
Cookies dapat di-set dari client

Men-set cookies dari server

Mengirimkan response header "Set-Cookie: key=value;"
Browser akan menyimpan-nya sampai browser tertutup
Untuk menghapus-nya, set "Expires" ke tanggal yang telah lewat dan "Max-Age" = 0

Expires dan Max-Age

Set-Cookie: id=1; Expires=Sat, 18 May 2024 00:00:00 GMT;
Cookie "id" akan kadaluwarsa pada 18 mei 2024
Set-Cookie: id=1; Expires=Mon, 13 May 2024 00:00:00 GMT; Max-Age=0
Cookie "id" telah terhapus

Domain dan Path

"Domain" dan "Path" merepresentasikan scope atau untuk siapa cookie tersebut diperuntukkan
Jika nilainya di-set, maka cookie tersebut hanya dapat diakses dari domain dan path yang telah ditentukan
Nilai default dari Domain adalah host, tidak termasuk subdomain
Nilai default dari Path adalah /, termasuk sub-path-nya
Set-Cookie: id=1; Domain: google.com;

Secure dan HttpOnly

"Secure" flag hanya dapat digunakan jika diakses melalui HTTPS
"HttpOnly" flag mengindikasikan jika cookie tersebut tidak dapat diakses melalui javascript
Set-Cookie: id=1; Secure; HttpOnly;

Men-set cookies dari client / javascript

Jika cookie dapat di-set melalui server dengan cara:
Set-Cookie: id=1; Expires=Sat, 18 May 2024 00:00:00 GMT; Path=/; Domain=*.google.com
Cookie juga dapat di-set dengan javascript di browser:
document.cookie = "id=1; Expires=Sat, 18 May 2024 00:00:00 GMT; Path=/; Domain=*.google.com"

Introduction: Json Web Token

https://jwt.io/
JWT adalah standar terbuka (RFC 7519) yang memungkinkan untuk berbagi informasi melalui encrypted JSON object
Dapat digunakan sebagai alternative dari cookie

Introduction: Localstorage and Sessionstorage

Alternative penyimpanan data di browser, termasuk token
Localstorage: tidak expired walaupun browser ditutup
Sessionstorage: expired saat browser ditutup
Menggunakan javascript untuk memanage data

Apa itu PHP Session?

Digunakan untuk menyimpan informasi sementara terkait suatu user yang dapat digunakan oleh seluruh halamn website
Informasi ini disimpan dalam sebuah file, atau database, atau cache server
Menggunakan cookie sebagai identifier: "PHPSESSID"

Cara kerja PHP Session

PHP server akan menginisiasi 32 digit random string sebagai PHPSESSID
PHP server akan membuat file, baris tabel, atau sebuah key di cache server yang relevan dengan PHPSESSID
Mengirimkan PHPSESSID sebagai cookie, yang akan melekat untuk setiap request

Cara menggunakan PHP Session


              session_start();

              $_SESSION["counter"] = 1;
              echo $_SESSION["counter"];
              uset($_SESSION["counter"]);

              session_destroy();

Konfigurasi keamanan

Fungsi session_start() dapat menerima parameter tambahan untuk memberikan keamanan tambahan
Parameter dapat dilihat pada php.ini atau phpinfo()

              session_start([
                'cookie_lifetime' => 14400,
                'cookie_secure' => true,
                'cookie_httponly' => true
              ]);
